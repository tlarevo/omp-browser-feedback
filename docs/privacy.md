# Privacy Disclosure

This document describes what browser data the OMP Browser Feedback system
captures, how it is transmitted, where it is stored, and how to revoke access.

## Data captured

### DOM selection feedback

When you click an element with the picker active, the Chrome extension captures:

| Field | Max size | Description |
|-------|----------|-------------|
| Element selector | — | CSS selector generated by the picker (prefers `data-testid`, `aria-label`, `id`, `name`) |
| Outer HTML | 20,000 chars | The element's serialized HTML |
| Tag name | — | Element tag (e.g. `button`, `div`) |
| Attributes | 80 max | All element attributes as key-value pairs |
| Computed styles | 80 max | Selected CSS properties from `getComputedStyle` |
| Accessibility tree | — | `role`, `name`, `description` from ARIA attributes |
| Viewport | — | Page dimensions, scroll position, device pixel ratio |
| Page context | — | URL, title, doctype |
| Screenshot | 10 MB max | Cropped around the element with 40px padding (or full visible tab as fallback) |
| Note | 8,000 chars | Optional annotation you typed before picking |

### Page screenshot feedback

When you capture a full-page screenshot (without element selection):

| Field | Max size | Description |
|-------|----------|-------------|
| Screenshot | 10 MB max | Full visible tab capture |
| Page context | — | URL, title, doctype |
| Note | 8,000 chars | Optional annotation |

## What is NOT captured

- **No browsing history** — only the current page when you click.
- **No form input values** — only element structure and attributes.
- **No cookies or session data** — the extension does not read cookie storage.
- **No cross-origin content** — same-origin policy applies; only the current page DOM is accessible.
- **No background tab data** — the `activeTab` permission only grants access when you click the extension icon.
- **No network requests** — the extension does not intercept or log HTTP traffic.

## Consent boundaries

1. **No data is captured until you act.** The picker overlay must be active and you must click an element.
2. **`activeTab` permission** grants access to the current tab only when you click the extension icon. No background monitoring.
3. **`scripting` permission** is used solely to inject the picker overlay on demand.
4. **`storage` permission** stores only: the broker discovery URL, the browser capability token, and the selected session ID. No browsing data.
5. **`tabs` permission** is used to capture screenshots via `chrome.tabs.captureVisibleTab`.

## Transport

All data flows over **loopback only** (`127.0.0.1`).

```
Chrome Extension ──HTTP/WS──> Broker (127.0.0.1:4317) ──WS──> OMP Extension
```

- **No external network calls.** Nothing leaves your machine.
- **Bearer-token authentication** on every non-health endpoint.
- **WebSocket** for real-time feedback delivery to the OMP session.
- **Origin checks** reject cross-origin requests to the broker.

## On-disk locations

| File | Location | Contents | Retention |
|------|----------|----------|-----------|
| Discovery file | `~/.omp/browser-broker.json` | Broker URL, auth token, PID, start time | Until broker stops or file is deleted |
| Pairing registry | `~/.omp/browser-pairing-registry.json` | Browser capability token hashes, labels, timestamps | Until `/bf pair reset` or file is deleted |
| Config file | `~/.omp/browser-feedback.json` | `autoRun` preference | Until file is deleted |
| Screenshots | `/tmp/omp-browser-screenshots/` | Cropped PNG/JPEG screenshots | Until broker shuts down |

## In-memory only (not persisted)

- **Feedback events** — stored per-session in the broker's in-memory store,
  bounded to 50 events per channel. Cleared when the broker stops.
- **Session registry** — active OMP sessions are tracked in memory only.
  Disconnected sessions remain in the registry until the broker restarts.

## Revocation

### Revoke a single browser

The Chrome extension's capability token is stored in `chrome.storage.local`.
Clearing the extension's storage or reinstalling it revokes that browser's access.

### Revoke all browsers

```
/bf pair reset
```

This immediately revokes all browser capability tokens. Every browser must
re-pair using a new code.

### Force a clean slate

```bash
rm ~/.omp/browser-pairing-registry.json
rm ~/.omp/browser-broker.json
```

This removes all pairing state and broker discovery. The next OMP session
starts fresh.

## Permissions justification

| Permission | Why it's needed |
|------------|-----------------|
| `activeTab` | Access the current tab's DOM when you click the extension icon |
| `scripting` | Inject the picker overlay into the page on demand |
| `storage` | Persist broker URL, capability token, and session selection across popup opens |
| `tabs` | Capture screenshots of the visible tab via `captureVisibleTab` |
| `host_permissions: http://127.0.0.1:*/*` | Communicate with the local broker on loopback |
